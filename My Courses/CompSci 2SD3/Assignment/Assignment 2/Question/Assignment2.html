Description of the C program assgn2.c
 
Since this is not a programming course, there are no partial marks or credits for having a correct header for a function or a correct variable etc. as you may be used to. The program will be tested solely on its performance, i.e. how well it performs the required tasks.
 
Your task is to write a multithreaded C program using POSIX  user threads. The program must be organized so that if a user wants to run your program with 8 personalities, the user will compile it with make assgn2 level=8 , if the user wants 3 personalities, the compilation will be make assgn2 level=3 . The permitted levels are 1 to 8.
The compilation will work for any level, it is during the execution when the level is checked, for instance:
make assgn2 level=10
will trigger the compilation
gcc -o assgn2 assgn2.c log.c -D_level=10 -D_multithread -lpthread
that will go through without a hitch, but when you try execute the program assgn2 , you get an error message
the program was compiled with a wrong level
permitted levels are 1 .. 8
and the program terminates right then.
 
The value of the level is determined during the compilation by the value of the _level flag. Thus, in your program you use anywhere you need it _level as if it were a varfiable; could be used even in the definition of the size of an array.
 
Each personality converses with the user and logs its conversation in a common log file whose name is assgn2.log . The conversation is simple, the personality displays the prompt, the user types in the response, so only the user is ``speaking". If the user's response starts with quit (case sensitive), the conversation is over, anything else is logged into the log and the thread continues. You have to use the log.c and log.h for this as they are thread-safe. Do not use the standard C file functions fopen() , fprintf() , and fclose() for processing the log.
 
Each personality has a permanent name: Person0, Person1, Person2 .... Person7 and a permanent index (Person0 has index 0, Person1 has index 1, Person2 has index 2 ...) Each personality has a variable linecount that is initialized to 0 .

#include "log.h"
The format of the prompt is given and you must adhere to it:

[pid=4196528,tid=1590835520,index=0,name=Person0] Enter Message:
where pid is the actual process id, tid is the actual thread id, index is the index of the Person , and name is the name of the Person
 
The high level structure of your program ought to be:
  1. It defines the global variables required for the logging functions, ie.
    int logindex = 0;
    int* logi = &logindex;
    // mutex for log, required by logging functions for thread-safe code
    pthread_mutex_t tlock = PTHREAD_MUTEX_INITIALIZER;
  2. It defines a global variable int done=0; that serves as the indicator when the processing is done
  3. It defines a mutex for done (the mutex is needed as all threads read and write to done)
  4. It defines a structure to hold all relevant information for a personality: index , the thread id tid , name of the personality, linecount , and flag indicating when the personality's processing is completed.
  5. It defines a table with as many entries as personalities, where each item is a structure defined just above.
  6. It first checks the _level and if it is not in the range 1..8, an error message is displayed and the program terminates.
  7. It opens the log with create_log("assgn2.log");
    Note that there is no need to protect the log by a mutex as the log.c code is thread-safe; and that there is no need to open and close the log -- it is all automatic.
  8. loop for i = 0 .. _level-1
        initialize the values on the table for for Person_i
    endloop
  9. do
        loop for i = 0 .. _level-1
            if Person_i not done yet, create a thread for it with pthread_create()
            include a test if pthread_create() worked, if not, display an error message and
            terminate the program with exit(1);
        endloop
        loop for i = 0 .. _level-1
            if Person_i not done yet, join the thread with pthread_join()
            include a test if pthread_join() worked, if not, display an error message and
            terminate the program with exit(1);
        endloop
    while     ! done
    Since done starts with value 0, the testing of its value must be at the bottom of the loop, not at the top of the loop
  10. Display how many lines each conversation had by traversing the table and using the linecount for each personality.
 
Each thread does the following
  1. Locks the mutex for the global variable done
  2. If the Person is just starting, it increments done by 1
  3. Displays the prompt
  4. Gets the users response
  5. Unlocks the mutex for done
  6. If the response started with quit, then
    • locks the mutex for the global variable done
    • decrements done by 1
    • unlocks the mutex for done
    • displays and logs a message that the user quit
    • exits the thread, its return value is irrelevant
  7. If the response did not start with quit, then
    • increments by 1 the linecount for this conversation
    • writes the message consisting of the prompt and the response into the log (for logging functions, you do not include the terminating \n to the message, however, if the message is a multiple-line, please follow any embedded \n with one space)
    • exits the thread, its return value is irrelevant
Note that a personality and a thread are not the same things. It may happen that for each line of the conversation, a personality is given a different thread. However, through the threads, the conversations are simultaneous
