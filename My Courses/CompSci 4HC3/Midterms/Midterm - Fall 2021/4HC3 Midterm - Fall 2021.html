Fitts' Law is used to calculate the time required to hit a target. Assume the constraints in the Fitts' Law equation for a trackball are a=75ms and b=300ms/bit.

a) (6 marks) Write the expression to determine how long it would take to move to the target with a trackball if the distance to the target is 35 pixels and the width (size) of the target is 2 pixels

b) (6 marks) Is it faster if the target distance is 40 pixels and the target size is 5 pixels? Explain your reasoning. A calculator is not required.

ANSWER:

There are two equations for Fitt's law. They are:

i. Movement time = a + b [log(2 * (d / s))]
ii. Movement time = a + b [log((d / s) + 1)]
(Note: The log is base-2)
(Note: `d` = distance)
(Note: `s` = size)

We will use Shannon's formulation (ii) for calculation

A)

a = 75ms
b = 300ms/bit
d = 35 pixels
s = 2 pixels

Movement time = a + b [log((d / s) + 1)]
Movement time = 75 + 300 [log((35 / 2) + 1)]
Movement time = 75 + 300 [log(17.5 + 1)]
Movement time = 75 + 300 [log(18.5)]
(Solving the equation yields)
Movement time ~ 1337.83599
Movement time ~ 1337.8

B) If the target distance is 40 pixels and the target size is 5 pixels, then the resulting movement time is faster/better. This is because increasing the size of the target greatly improves accuracy by several magnitudes. In this case, the target increased by a magnitude of 2.5, but the distance increased by 5 pixels. And according to the Fitt's law equation, since `s` is under `d`, it dictates the size of `d/s`. Hence, a large `s` value will decrease the movement time. Therefore, the movement time decreases, which makes it faster to reach the target.

Solving the equation gives us:

Movement time = a + b [log((d / s) + 1)]
Movement time = 75 + 300 [log((40 / 5) + 1)]
Movement time = 75 + 300 [log(8 + 1)]
Movement time = 75 + 300 [log(9)]
Movement time ~ 1025.9775
Movement time ~ 1026.0

(6 marks) Imagine a research project in which we want to investigate technology to assist in meal preparation. In designing a questionnaire to gather data about how people currently prepare meals, you realize that this question is problematic

  • How many hours did you spend preparing meals yesterday

Why is the question inappropriate for a questionnaire? Suggest a way to rewrite it to obtain more reliable responses

ANSWER:

Q: Why is the question inappropriate for a questionnaire?
A: The question is inappropriate for a questionnaire because it is too specific. The question asks users to specifically recall how much time they spent preparing meals a day ago. Users not may accurately recall how much time they spent on meal prep. In addition, the timeline - or range - of the question is way too narrow. For instance, what if the user prepared all their meals at the beginning of the week, like Sunday, and spend very little or no time at all in preparing meals during the weekdays. So, regardless of when this question is asked, the answer will be very misleading.

Also, the questionnaire should specify what counts as preparing meals? For instance, some users may consider re-heating meals cooked earlier as meal prep. In addition, what if some users do not prepare meals themselves, but have someone else do it for them? What if users order take-out or dine-in? Scenarios like these should be accounted for - maybe not in this question, but somewhere in the survey or questionnaire.

Q: Suggest a way to rewrite it to obtain more reliable responses?
A: To obtain more reliable responses, the question should:
- Use words like "think", "reflect", "recall", etc. This is because language is important. Setting the tone, or a tone, makes a big difference in how users perceive questions and how they'll answer them.
- Increase the time window. For example, instead of asking about yesterday, the question should ask about average time spent on preparing meals in a week. However, asking users to recall their weekly events may stress their memory too much, but the results will be more accurate than asking them about a narrow time window, like yesterday. This is especially important if we are building technology to help with meal prep. For example, is the network infrastructure going to be stressed daily, or will most of the network stress come on a particular day, like Sunday.

One way to reword the question is:

"Recall how much time you spend, on average, per day, on preparing meals. Please factor in all the time you spend preparing meals in a week, and not just how much time you spent yesterday preparing meals."

(6 marks total) Compare and contrast the command line interaction style with the WIMP interaction style. (4 marks) What is each good for or not good for? (2 marks) Define the components of WIMP in your answer

ANSWER:

CLI = Command Line Interface
WIMP = Window, Icon, Menu, Pointer
GUI = Graphical User Interface

(Note: In the context of this question, a GUI is not the same thing as WIMP).

Q: Compare and contrast the command line interaction style with the WIMP interaction style.
A: The command line interaction (CLI) style is very different from the WIMP interaction style. Both interfaces look, and feel different, and serve two different kinds of users. The CLI is very barebones. Often times it is just a black box (called a terminal), with text across it. Inside this terminal, the user types actions and executes commands. Input is primarily done via a keyboard or any other device that allows for inputting ASCII-like characters. After every command, the user needs to press {ENTER}, on the keyboard, to execute it. In a CLI, the user needs to memorize the commands, and know how to read the manual pages for each command. Help is very limited, and it's not obvious/intuitive what needs to be entered or executed. On the other hand, the WIMP style of interaction is very intuitive, relative to a CLI. In WIMP, there are windows, icons, and menus. These elements can be interacted with a pointer, or even a keyboard. The appearance or design of a WIMP can heavily vary depending on the design team; WIMP affords more flexibility in terms of design. Furthermore, in WIMP, different colors, styles, patterns, etc. can be utilized to create a unique experience. This is not, or less, true for CLI. More information about WIMP can be found below.

Q: What is each good for or not good for?
A: CLI is great for advanced, or power, users. These users know what commands to use, how to use them, and when to use them. CLI is not good for novice or new users. This is because the design of CLI is not intuitive nor is it beginner friendly - it has a steep learning curve for new users. However, productivity is higher with CLI than with WIMP. This is because the barebones style of CLI allow users to get more done with limited distractions. In addition, CLI allows users to automate actions. For example, the ".bashrc" file in a Unix system contains shortcuts for lengthy commands. Since a CLI is barebones, it can be deployed on limited or low-end hardware like embedded devices, such as the one's in your toaster/oven. On the other hand, WIMP afford a high degree of change-ability. Almost anything, design wise, can be changed in a WIMP system. In addition, this allows for WIMP to be deployed on lots of different systems, and it can be tailored to suite certain individual's needs - for example, disabled people. If the design of a WIMP system is intuitive, it can be much easier to learn than CLI, but if the WIMP interface is poorly designed then it can be harder to learn. A downside to WIMP is that it is hardware-hungry. WIMP interfaces may not work smoothly on older hardware.

Q: Define the components of WIMP
A: WIMP stands for Window(s), Icon(s), Menu(s), Pointer(s). More information is below.

Windows are independent applications. For example, calculator has its own window, and minesweeper has its own window, and the browser has its own window, and the file management system has its own window as well. Windows are also used to display dialog boxes which can be used for notifying the user of an error, or asking the user for some input.

Icons represent things like applications, interact-able objects, etc. For example, the calculator application may have an equals sign (=) as its icon. The file management system may have a binder/folder as its icon. The music application may have an octave symbol as its icon. Icons make it easy for the user to tell what application is what, or what an action may or may not do.

Menu(s) can be dropdown or brought up with a click, left or right. Typically, menus present more options that afford the user more functionalities. For example, a menu that is embedded inside an application like Minesweeper may allow the user to change the size of the board, toggle hints, change colors, reset the game, etc.

Pointers are used to interact with the other elements of WIMP. A pointer can be a device like a mouse, trackpad, touchpad, stylus, etc. The user can move the pointer to a desired location to interact with an element of WIMP. For instance, a user may move the pointer to an Icon, and then double-click it to execute the application/command.
