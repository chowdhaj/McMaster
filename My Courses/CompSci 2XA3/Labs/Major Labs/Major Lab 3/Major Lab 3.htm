Task 1. make file named makefile1
The name of your file must be makefile1 and below is a description of what it should do when executed. Note that the names of the make files, the scripts, and of the files and directories, and the messages must be used exactly as described here, including the lower and upper cases.
 
Before you start working on the make file makefile1:
  • Decide what will be your working directory.
  • Download a text file fileA and save it on your workstation, then transfer it to moore to your working directory and convert it to a unix text file (using dos2unix). This file is necessary for the make file to work.
  • Download a text file fileB and save it on your workstation, then transfer it to moore to your working directory and convert it to a unix text file. This file is necessary for the make file to work.
  • Download a text file fileC and save it on your workstation, then transfer it to moore to your working directory and convert it to a unix text file. This file is necessary for the make file to work.
  • Download a C++ program yous1.cpp and save it on your workstation, then transfer it to moore to your working directory and convert it to a unix text file. This program is necessary for the make file to work.
  • Download a bash script script1 and save it on your workstation, then transfer it to moore to your working directory and convert it to a unix text file. Then make it executable (using chmod). This script is necessary for the make file to work.
 
What should makefile1 do:
  1. The make file is to be located in the working directory.
  2. It creates a bash script named script2 from script1 by replacing every occurrence of a character 1 in the file script1 by a character 2 , and makes script2 executable using chmod command.
  3. It creates a bash script named script3 from script1 by replacing every occurrence of a character 1 in the file script1 by a character 4 , and makes script3 executable. (Note that it replaces 1 by 4 and not by 3 !)
  4. It creates a C++ program (file) named yous.cpp by concatenation of the contents of the three files fileA, fileB, and fileC (in that order).
  5. It creates a C++ program (file) named yous2.cpp from yous1.cpp by substituting every occurrence of a character 1 in yous1.cpp by a character 2 .
  6. It creates a C++ program (file) named yous3.cpp from yous1.cpp by substituting every occurrence of a character 1 in yous1.cpp by a character 3 .
  7. It creates a C++ program (file) named yous4.cpp from yous1.cpp by first substituting every occurrence of a character 1 in yous1.cpp by a character 4 producing a temporary file. Then each occurrence of a word FUNCTIONAL in the temporary file is replaced by a phrase NOT FUNCTIONAL producing yous4.cpp .
  8. It creates object files yous1.o, yous2.o, yous3.o, and yous4.o by compilation using the g++ -c , for instance g++ -c yous1.cpp to create yous1.o .
  9. It creates an executable file file1 from yous.cpp by executing the script script1 .
  10. If creates an executable file file2 from yous.cpp by executing the script script2 .
  11. It creates an executable file file3 by compilation of yous.cpp and linking of yous1.o, yous2.o, yous3.o, and yous4.o together by the g++ compiler with -D_X3 flag, i.e. g++ -o file3 yous.cpp -D_X3 yous1.o yous2.o yous3.o yous4.o .
  12. It creates an executable file4 from yous.cpp by executing the script script3 .
  13. When typing make -f makefile1 all , the executables file1, file2, file3, and file4 must be created.
  14. When you execute file1, you should see a message my breakfast was great
  15. When you execute file2, you should see a message my breakfast was lousy
  16. When you execute file3, you should see four messages
    system X1 is FUNCTIONAL
    system X2 is FUNCTIONAL
    system X3 is FUNCTIONAL
    system X4 is NOT FUNCTIONAL
  17. When you execute file4, you should see a message X not specified
  18. When typing make -f makefile1 clean , all created files in the working directory must be removed and the directory must only contain file1, file2, file3, yous1.cpp and script1 as at the beginning (Do not remove makefile1 nor makefile2 !)
 
A few useful hints:
  • If you do not have the current directory in the path, you have to refer to the files in your directory in the make file with the prefix ./ , for instance cat ./xxx instead of cat xxx . To quickly add the current directory to the path, execute the commands echo "PATH=$PATH:." >> ~/.bashrc and then source ~/.bashrc
  • A single letter can be "translated" by tr command: for instance tr "1" "2" < fin > fout will read the input file named fin and output it to a file named fout and during this process it will translate each character 1 to 2 . The file fin will remain unchanged, the changes will be in the file fout .
  • A substring (of any length, including a substring of length 1, i.e. single letter) can be "translated" to any other string by sed command, for instance sed 's/HELLO/HELLO BYE/' fin > fout will read the input file fin and write into the output file fout while replacing the first occurrence of HELLO in each line with HELLO BYE . The file fin will remain unchanged, the changes will be in the file fout .
 
Task 2. make file named makefile2
The name of your make file must be makefile2 and below is a description of what it should do when used.
 
Before you start working on makefile2:
  • Decide what will be your working directory.
  • In the working directory, create a subdirectory PI1 with a file text.asc downloaded from here, using dos2unix make sure it is a unix text file. You will need this directory and this file only for testing.
  • In the working directory, create a subdirectory PI2 with a file text.asc downloaded from here, make sure it is a unix text file. You will need this directory and this file only for testing.
  • In the working directory, create a subdirectory PI3 with a file text.asc downloaded from here, make sure it is a unix text file. You will need this directory and this file only for testing.
  • In the working directory, create a subdirectory PI4 with a file text.asc downloaded from here, make sure it is a unix text file. You will need this directory and this file only for testing.
  • In the working directory, create a subdirectory PI5 with a file text.asc downloaded from here, make sure it is a unix text file. You will need this directory and this file only for testing.
  • Download a C++ program format.cpp and transfer it to moore to the working directory and make sure it is a unix text file. This program is essential for making the make file work.
What should makefile2 do:
  1. There are private investigators writers, PI1, ..., PI5 . Each of them keeps his/her notes in a file text.asc . The first PI keeps his/her notes in the directory PI1, the second PI keeps his/her in the directory PI2, ... , the fifth PI keeps his/her notes in the directory ( you guessed it :-) ) IP5 .
  2. But their notes are sloppy, so before their notes may be compiled together, their notes must be formatted by a program format . The executable format is obtained from the source file format.cpp by the standard C++ compilation: g++ -o format format.cpp . The program format expects the relative pathname of the file to be formatted as the first command line argument. For instance, to format the file text.asc from directory IP1 , you would use format IP1/text.asc . The formatted text is displayed on the standard output, i.e. on the screen. Note that the input file text.asc remains as it was, the formatted file is only displayed!
  3. The formatted texts are are all concatenated together (in the order 1 to 5) forming a file called report .
  4. At the beginning, the working directory only contains IP1/text.asc, IP2/text.asc, ... , IP5/text.asc and format.cpp and makefile2 . Prepare the make file makefile2 so that when make -f makefile2 report is executed, the notes of the 5 PIs are formatted, concatenated together, producing report . The make file can create temporary files, but they all must be removed before the make file is done, the same applies to the executable format -- it may not remain in the directory. So, after running make -f makefile2 report , only the file report is added to the content of the working directory ((Do not remove makefile1 nor makefile2 !). The text in report should consists of single sentences separated by a single empty line.
